Studi Kasus: Sistem Penjualan E-Commerce Shopee

1. Latar Belakang Shopee

Shopee adalah platform e-commerce yang menghubungkan jutaan pembeli dan penjual. Dengan skala transaksi yang masif, sistem memerlukan arsitektur yang terstruktur agar pengelolaan data produk dan transaksi berjalan efisien.

2. Masalah tentang Shopee

  • Ketidaksiapan sistem dalam menangani sinkronisasi stok secara real-time saat flash sale.
  • Kompleksitas pemeliharaan kode saat penambahan fitur baru di sistem yang sudah besar.
  • Kesulitan klasifikasi berbagai tipe produk (Elektronik, Fashion, dll) dalam satu alur transaksi.

3. Mengapa Masalah Ini Lebih Baik Diselesaikan dengan OOP?

Pendekatan OOP sangat ideal karena:

  • Inheritance: Memungkinkan pembuatan kelas produk umum yang diturunkan ke produk spesifik tanpa menulis ulang kode.
  • Enkapsulasi: Menyembunyikan logika transaksi yang sensitif dari akses luar.
  • Abstraksi: Menyederhanakan kompleksitas sistem menjadi objek-objek yang mudah dikelola.

4. Langkah-Langkah Menyelesaikan Masalah

  1. Analisis: Mengidentifikasi objek utama (User, Product, Cart, Transaction).
  2. Desain Kelas: Mendefinisikan atribut dan metode untuk tiap objek.
  3. Modularisasi: Membagi sistem ke dalam 5 file terpisah untuk menjaga kebersihan kode.
  4. Implementasi: Menghubungkan antar objek (misal: Cart menggunakan objek Product).
  5. Pengujian: Memvalidasi alur transaksi dari keranjang hingga pembayaran.

Penjelasan Singkat Source Code 1 - 5

  • Code 1 (product.dart): Kelas abstrak sebagai fondasi/blueprint setiap produk.
  • Code 2 (electronic.dart): Implementasi pewarisan (inheritance) untuk produk spesifik.
  • Code 3 (cart.dart): Logika untuk menampung daftar belanjaan.
  • Code 4 (user.dart): Manajemen data profil pembeli.
  • Code 5 (transaction.dart): Eksekusi akhir penggabungan seluruh objek dalam proses pembayaran.
